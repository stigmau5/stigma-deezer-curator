# Responsive Workspace Layout

Sprint Y shifts the Archive and Library tabs from fixed stacked panels toward resizable workspaces.

## Goals

The layout is designed to remain usable on common desktop sizes:

- 1366x768
- 1920x1080
- 2560x1440

The application still uses the filesystem and derived registries as its data sources. This sprint does not add playback, metadata fetching, SQLite, or archive mutation.

## Pane Hierarchy

### Archive Tab

The Archive tab uses a horizontal pane layout:

1. Archive Tree
2. Album List
3. Album Workspace

The Album Workspace receives the largest default weight because it contains the album evidence surface.

### Library Tab

The Library tab uses a horizontal pane layout:

1. Artist List
2. Album List
3. Album Workspace

The workspace is shared in spirit with the Archive tab, but Library remains lifecycle-oriented while Archive remains filesystem-oriented.

## Album Workspace

The album workspace uses a vertical pane hierarchy:

1. Top: cover art and archive status
2. Middle: overview, metadata, and identity
3. Bottom: tracklist and NFO evidence

The bottom evidence area receives the most growth. Tracklist and NFO are independently scrollable and can be resized with a horizontal pane boundary.

## Artwork Behavior

The cover panel has a fixed minimum display area so it does not collapse during resizing. Artwork is scaled down to fit the panel. Missing artwork remains visible as a centered placeholder.

## Layout Persistence

Window geometry and key pane positions are stored in `data/audio_division_settings.json` under the `ui` section.

Stored values include:

- `window_geometry`
- `archive_main_panes`
- `archive_workspace_panes`
- `archive_evidence_panes`
- `library_main_panes`
- `library_workspace_panes`
- `library_evidence_panes`

Pane positions are UI preferences only. They are not archive state and can be regenerated by deleting the settings values.

## Future Expansion

The same pane model can later support:

- playback controls
- richer local artwork previews
- NFO viewing modes
- operation result panels
- campaign or batch-operation views

Those future additions should remain subordinate to the album workspace instead of turning the interface back into a dashboard-first layout.
